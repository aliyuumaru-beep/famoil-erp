=======================================================
AI OPERATING RULES
=======================================================

1. Never push directly to main. All changes go through a PR and CI checks.

2. Never modify core Odoo files under odoo/.

3. Never expose passwords, API keys, database credentials, or secrets in output.

4. Never delete committed files without explicit operator approval.

5. Never proceed to the next implementation priority without operator approval.

6. Always take a backup before making changes:
   bash scripts/backup_famoil.sh

7. Always update CLAUDE.md after any significant change.

8. When unsure: STOP and ask the operator. Never take a destructive action
   to resolve ambiguity.

9. All implementation activity must comply with IMPLEMENTATION_STANDARDS.md
   and ARCHITECTURAL_PRINCIPLES.md.

10. The current active priority is defined in:
    docs/roadmap/PLATFORM_ROADMAP.md — Section 7 (Current High-Priority Next Steps)
    Current phase: Procurement Maturity & Operational Workflow Expansion

=======================================================
GOVERNANCE ENGINE
=======================================================

This repository has an active two-layer governance engine:

Layer 1 — Session Hooks (.claude/settings.json):
  Blocks dangerous bash commands (pre_tool_guard.sh)
  Blocks writes outside the repo (file_protection_guard.sh)
  Validates written files (post_tool_validator.sh)
  Logs all commands (audit_logger.sh)
  Loads context at session start (session_start_loader.sh)
  Reports session summary (session_end_reporter.sh)

Layer 2 — GitHub Actions (.github/workflows/):
  ci_review.yml     — Claude governance review on every PR
  doc_lint.yml      — documentation completeness check
  security_scan.yml — secret and credential scan
  backup_check.yml  — weekly backup currency validation
